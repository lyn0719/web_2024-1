주제 보수 표현과 계산 구체적인 주제 선정 이유 컴퓨터의 산술 연산은 왜 1의 보수가 아닌 2의 보수를 이용하여 계산하는 것일까하는 주제에 대한 궁금증 확대 개인 탐구 내용 -보수 : 두 수의 합이 진법의 밑수(N)가 되게 하는 수 -컴퓨터는 n개의 비트를 이용해 2^n개의 정수만 표현이 가능하다. 또한 정수는 양의 정수(+), 음의 정수(-), 0로 표현이 가능하다. 컴퓨터에서 이러한 +, - 부호를 표현하기 위해서도 비트로 표현 가능해야한다. -컴퓨터는 사칙연산을 할 때 덧셈을 할 수 있는 가산기(Adder)를 이용한다. 즉, A-B의 계산식에서는 B를 보수를 이용하여 –B로 변환을 한 후 A+(-B)로 계산을 해야 한다. -0을 표현하는 경우 : 1) 1의 보수 ; 0000, 1111 ⟹ 둘 다 0의 값을 가진다. = 0이 두 개가 존재하는 모순이 발생한다. 2) 2의 보수 ; 0000 ⟹ 0의 값을 가진다. 이에 따른 한 바이트(8비트)당 수의 크기는 1-1) 1의 보수 ; -127~127 ⟹ 총 255개 2-1) 2의 보수 ; -128~127 ⟹ 총 256개 이처럼 2의 보수를 취하면 1개의 수를 더 표시할 수 있게 된다. 결론적으로 1의 보수보다 2의 보수가 조금 더 효율적으로 산술을 할 수 있는 이점을 가지고 있게 된다. 탐구를 통해 느낀점 컴퓨터에서 음수를 표현하기 위해 보수 표현을 사용한다는 것이 신기했다. 1의 보수와 2의 보수에 대해 탐구해볼 수 있는 시간을 가져서 좋았다. 음수를 표현하기 위해서는 2의 보수로 표현하는 것이 왜 더 효율적인 것인지에 대해 자세히 알 수 있어 궁금증을 해결한 것 같아서 만족했다. 주제 상태 레지스터 - 오버플로우 구체적인 주제 선정 이유 레지스터 – 상태 레지스터에 포함된 오버플로우와 캐리의 차이를 알고 그 둘을 구분할 수 있도록 하기 위해서이다. 개인 탐구 내용 -상태 레지스터 : 상태 레지스터(status register)는 프로세서를 구성하는 레지스 터 중에 하나로 프로세서의 상태나 연산 결과의 상태를 저장하 기 위한 특수 목적의 레지스터이다. -상태 레지스터의 목적 : 덧셈, 뺄셈 연산에서 올림과 빌림이 발생됨을 알리기 위해 쓰인다. 올림과 빌림이 발생했다는 사실은 연산 결과로 알기 어렵기 때문이다. -오버플로우 : 오버플로우(Overflow)는 연산 결과가 해당 시스템에서 표현 가능 한 숫자 범위를 넘어가는 경우 발생하는 오류이다. -캐리 : 캐리(Carry)는 최상위 비트에서 그 위의 비트로 자리올림이 발생하는 경 우이다. -오버플로우와 캐리 : 캐리는 부호 있는(signed) 연산에서는 의미가 없고, 부호 없는(unsigned) 연산에서 의미가 있다.. 오버플로우는 (양수) + (양수) 또는 (음수) + (음수) 에서만 발생한다. (양수) + (음수)나 (음수) + (양수) 에서는 오버플로우가 발생하지 않는다. 또한, 오버플로우는 부호 없는(unsigned) 연산에서는 의미가 없다. 탐구를 통해 느낀점 이번 탐구를 통해 오버플로우와 캐리에 대해 더 자세히 탐구할 수 있었어서 좋은 기회였다. 상태 레지스터에서 오버 플로우와 캐리를 구분하는 데 있어 조금 헤메는 부분이 있었는 데 이번 활동을 통하여 오버플로우와 캐리를 구분하기 수월해 진 것 같아서 좋은 활동이였던 것 같다. 주제 운영체제의 종류 구체적인 주제 선정 이유 운영체제의 종류 안에 있는 다양한 운영체제들의 특징들을 탐구하고 싶어졌다. 개인 탐구 내용 -서버용-Unix 서버 : 운영 체제가 대부분 고급 언어인 C언어로 쓰여져 있고 소스 코드를 쉽게 구할 수 있어서 다른 컴퓨터 하드웨어나 새로운 기종에 적은 노력으로도 쉽게 이식할 수 있다. -서버용-Linux 서버 : 리눅스 OS 커널 소스코드에 GPL 라이선스를 채택하고 있다. 리눅스 OS는 무료 또는 유료로 판매/배포가 가능하다. 소스코드를 약간 수정하여 재배포/재판매도 가능하다. -서버용-Windows 서버 : Windows Server 운영체제에는 웹사이트를 운영하는 강력한 도구인 IIS(Internet Information Service), 네트워크에서 IP 풀을 관리 및 자동 할당해주는 DHCP, 떠오르는 가상화(가상 컴퓨터) 도구 Hyper-V. 심지어는 Linux에서 Cron으로 불리는 예약 작업 기능인 작업 스케줄러나, 서버를 백업할 수 있는(증분 백업까지도 가능합니다!) Windows Server 백업과 같은 강력한 기능을 포함하고 있다. Windows Server 운영체제를 설치하는 것만으로 서버에서 필요한 거의 모든 소프트웨어가 따라 들어온다. -데스크탑용-Window : Microsoft에서 개발하는 컴퓨터 운영체제. 컴퓨터 역사상 가장 대중적으로 널리 알려지고, 가장 많이 사용되는 운영체제이다. -데스크탑용-Mac OS : 모토로라 680x0나 PowerPC CPU[4]를 탑재한 구형 Apple Macintosh 컴퓨터에서만 돌아간다. 응용 프로그램 설치가 간단하다. ㄱ드라이버 설치가 간단하다. 운영 체제 재설치가 간단하다. 확장자가 없다. -모바일용-Android : Android는 리눅스 커널을 기반으로 Google에서 제작한 스마트폰과 같은 플랫폼의 모바일 운영 체제와 미들웨어 및 중요 애플리케이션이 포함된 소프트웨어 집합이다. 2019년 Android는 세계에서 가장 대표적인 오픈 소스 플랫폼이며 세계 최다 사용자를 보유한 운영 체제다. -모바일용-iOS : iOS는 macOS를 기반으로 만들어져 있다. macOS와 커널 구조가 상당히 유사하며 리눅스나 Unix-like OS 등과 시스템 구조가 비슷하다 탐구를 통해 느낀점 다양한 운영체제들의 특징에 대해 탐구해볼 수 있는 시간을 가질 수 있어서 흥미로웠던 시간이였다. 여러 운영체제들의 이름은 알지만 정확하게 어떤 특징들을 가지고 있는지는 몰라서 이번 탐구를 통해 각각의 운영체제들에 대해 조금 더 자세히 알 수 있게 되어서 좋았다.
